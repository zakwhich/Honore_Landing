# Project Coding Instructions

## To Do Workflow

At the start of every task, check for a folder named `To Do`.

Before performing any coding work for a feature or fix, create a careful overview file in `To Do` that describes the work to be completed. Use this filename format:

`MMDDYY (FIX/FEATURE) Short Name or Description (Incomplete/Complete).md`

The filename must contain both parenthetical labels:

- `(FIX)` or `(FEATURE)` to identify the type of work.
- `(Incomplete)` or `(Complete)` to identify its current status.

At the top of the overview file, include a `Last Updated` date and time.

The overview file must include:

- A clear overview of the intended feature or fix.
- Scope boundaries and relevant implementation decisions.
- Work divided into sensible phases.
- A checklist of coding and verification items within those phases.

As work is completed:

- Check off each completed checklist item.
- Update the `Last Updated` date and time whenever the file is changed.
- Keep the overview accurate if the implementation plan changes.

When all work is implemented and verified:

- Add a short completion overview near the top describing what was done.
- Update the `Last Updated` date and time.
- Rename the file so its final status label changes from `(Incomplete)` to `(Complete)`.
- Keep only one task file for each feature or fix. Completing a task must replace the `(Incomplete)` file with the `(Complete)` file, usually by renaming/moving it. Do not leave both an incomplete and complete version of the same task in `To Do`.
